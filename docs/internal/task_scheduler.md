# jogasaki ジョブ・タスクスケジューラ内部設計メモ

2025-03-28 kurosawa

## この文書について

jogasakiのタスクスケジューラの内部設計について設計上の特筆すべきトピックをメモとして残す

## 用語

- タスク

  - シングルスレッドで実行可能な細粒度の処理の単位となるもの
  - `jogasaki::scheduler::flat_task` クラスで表される
  - jogasakiのプランスケジューラ(dag_controller)でもタスクが現れるがそれを含むより広い概念
 
- ジョブ

  - 1個以上のタスクからなる
  - 管理領域として `jogasaki::scheduler::job_context` クラスで表される
  - `teardown` タスクによって待ち合わせ

- リクエスト



- タスクスケジューラ
 
  - タスクの実行を計画し、CPUリソースを割り当てて適切なタイミングで実行するもの 
  - タスクスケジューラへタスクを渡して計画させる処理を「スケジュール」という
  - タスクスケジューラが適切なスレッド上でタスクを開始し完了させることを「実行」という

- ワーカー

## タスクスケジューラの機能

- スレッドプール

  - スレッドをプールし必要に応じてタスクにアサインして再利用する
  - スレッドの開始・停止によるコストを避ける
  - 使用スレッドの個数が際限なく増えることを避ける (使用スレッド数は起動開始時から停止まで一定)
  - 各スレッドを特定のCPUコアにバインドし、コアに対する複数スレッドの取り合いを避ける

- 同一トランザクションを使用するタスクの整流

- スティーリングによる空きCPUコアの活用、負荷分散
- 
- サスペンド・復帰

- 条件確認タスク
